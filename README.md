## ОСНОВЫ HTML
HTML (HyperText Marked Language) - язык разметки, который на данный момент использует 5 версию. \
Существует основной шаблон, который используется при создании HTML файла
```HTML
<!DOCTYPE html>
<html>
	<head>
		<meta charset = "utf-8">
		<meta name="viewport" content="width=device-width, initial-scale=1">
		<title>Название вкладки</title>
		<link rel="stylesheet" type="text/css" href="style.css">
		<script src="script.js"></script>
	</head>
	<body>
    
  </body>
</html>
```

Давайте теперь разберем что написано в данном щаблоне. Первая строчка указывает на то, что мы используем HTML именно 5 версии.\
4 строчка при помощи атрибута `charset` подключает кириллицу.\
В 5 строчке мы указываем параметры отображения контента на нашем экране. `content="width=device-width, initial-scale=1"` - ширина по умолчанию равна ширине экрана, а масштаб по умолчанию будет 100%.\
В 6 строчке при помощи тега `<title>` мы задаем название вкладки, которое будет отображаться в браузере.\
7 и 8 строчки ответственны за подключение файлов формата CSS и JS.\
Всё, что касается настройки вашего сайта, прописывается в теге `<head>`. А само содержимое сайта уже в теге `<body>`.

### ОСНОВНЫЕ ТЕГИ HTML
`<H1>`,`<H2>`,...,`<H6>` - теги для заголовков.\
`<H1>` - самый главный заголовок. А `<H6>` - самый маленький.\
Для абзацев следует использовать тег `<p>` - При его открытии и закрытии всё что написано будет отображаться с новой строки. Пример:
```HTML
<p>Какой-то текст</p>
<p>Новая строка</p>
```
Если же вы хотите просто сделать перенос на новую строку, то следует использовать одиночный тег `<br>`.\
Для выделения текста курсивом, жирным или подчеркиванием нужно использовать парные теги `<i>`,`<b>` или `<u>` соответственно.
```HTML
<p>Весь текст, который находится внутри парных тегов, будет форматирован как <b>жирный</b>, <i>курсив</i> или <u>подчеркнутый</u></p>
```
Если вам нужно вставить ссылку на ваш сайт, то следует использовать следующую конструкцию:
```HTML
<a href="https://developer.mozilla.org/ru/docs/Learn/Getting_started_with_the_web"
  >Cайт, который поможет вам в изучении HTML, CSS и JS</a>
```
При помощи атрибута `href=" "` мы указываем ссылку, куда будет осуществлен переход.\
А всё, что находится внутри теги `<a>`, будет являться переходом. В нашем случае это текст.

#### СПИСКИ
HTML поддерживает несколько видов списков: неупорядоченный (маркированный) `<ul>` - unordered list, упорядоченный (нумерованный) `<ol>` - ordered list и список определений `<dl>` - list of definition.\
При этом для упорядоченнного списка вы можете также задать вид маркера при помощи атрибута `type=""`. По умолчанию принимает значение `1`.\
Пример:
```HTML
<ul>
   <li>Первый пункт</li>
   <li>Второй пункт</li>
   <li>Третий пункт</li>
</ul>

<ol>
   <li>Первый пункт</li>
   <li>Второй пункт</li>
   <li>Третий пункт</li>
</ol>

<ol type="A">
   <li>Первый пункт</li>
   <li>Второй пункт</li>
   <li>Третий пункт</li>
</ol>

<ol type="I">
   <li>Первый пункт</li>
   <li>Второй пункт</li>
   <li>Третий пункт</li>
</ol>

<dl>
  <dt>Первый термин</dt>
    <dd>Его определение</dd>
  <dt>Второй термин</dt>
    <dd>Его опредление</dd>
    <dd>Еще одно определение</dd>
</dl>
```
Также вы можете делать вложенные списки, если это потребуется

#### ТАБЛИЦЫ 
В HTML без использования стилей вы также можете создать таблицы. Делается это при помощи тегов `<table>`, `<tr>` и `<td>` - создание таблицы, строчки и ячейки соответственно.\
Пример:
```HTML
<table>
   <tr>
      <td>Ячейка 1.1</td>
      <td>Ячейка 1.2</td>
      <td>Ячейка 1.3</td>
   </tr>
   <tr>
      <td>Ячейка 2.1</td>
      <td>Ячейка 2.2</td>
      <td>Ячейка 2.3</td>
   </tr>
</table>
```
Если вы хотите растянуть какую-нибудь ячейку, то делается это при помозщи атрибутов `colspan=""` и `rowspan=""`.

Для комментариев в HTML используется следуюйщая конструкция: `<!--Комментарий-->`.\
HTML также способен создавать кнопки, поля ввода, изображения, аудио- и видеоконтент.\
Для этого используются теги `<button>`, `<input>`, `<audio>` и `<video>`.\
Пример:
```HTML
<button>Нажми меня</button> <!--Внутри теги пишется текст, который будет отображаться на вашей кнопке-->

<input type="text" value="Введите пароль"> <!--в value указывается то, что будет находится в поле изначально-->

<audio controls> <!--controls подключает управление вашим контентом-->
  <source src="audio.mp3" type="audio/mpeg"> <!--при помощи src вы задаете путь до вашего файла, а при помощи type указываете тип файла-->
    Браузер не поддерживает данный формат. <!--Если браузер не поддерживает формат файла, то вы можете дать ссылку на скачивание файла-->
    <a href="audio.mp3">Скачать трек</a>
</audio>

<video src="video.mp4" poster="poster.jpg"><!--при помощи src вы задаете путь до вашего файла, а poster - изображение до начала воспроизведения-->
  Ваш браузер не поддерживает данный формат. 
  <a href="video.mp4">Скачать видео</a>
</video>
```
Для тега `<input>` вы можете менять типы ввода и изменять размеры текстового поля.\
Для `<video>` вы также можете добавлять субтитры к видео.\
Ранее в браузерах также можно было задавать автоматическое воспроизведение контента, но данная функция теперь запрещена, так как мешает пользователям.

## ОСНОВЫ ФОРМАТИРОВАНИЯ ТЕКСТА
Есть несколько разных способ взаимодействия со стилями в HTML.\
\
**Первый** заключается в прописывании элементов стилей как атрибутов внутри самого тега. Проблема этого метода состоит в том, что мало стилистических свойств поддерживают данный метод и существует необходимость прописывать стили при каждом новом объявлении тега.\
\
**Второй способ** - использование фрагмента CSS кода внутри HTML при помощи тега `<style>`. Этот способ в разы лучше, а иногда он даже является наилучшим вариантом, если у вас существует всего несколько строчек стилей и для этого нет необходимости создавать новый файл.\
\
**Третий способ** - создание отдельного CSS файла. Это самый верный способ. В HTML вы храните только ссылку на данный файл, а всё что касается изменение стилей находится в другом файле и не захламляет код разметки страницы.\
\
Пример:
```HTML
<!--Первый способ через атрибут style=" ".-->
<p style="font-size:24px; color: red">Hello World</p> 

<!--Второй способ через тег <style>.-->
<style>
   p{
      font-size:24px;
      color:red;
   }
</style>

<!--Третий способ через ссылку на CSS файл. Дизайн прописывается аналогично тегу <style>.-->
<link rel="stylesheet" type="text/css" href="style.css">
```
Давайте договоримся, что в данном курсе все стилистические махинации мы будем производить в отдельном CSS файле. Поэтому третий вариант для нас является образцовым.\
\
Комментарии в CSS оставляются при помощи следующей структуры `/*Комментарий*/`.\
\
Попробуем теперь изменить отображение текста на всей странице. Вы можете поменять следующие параметры: шрифт (`font-family`), размер текста (`font-size`), толщину обводки (`font-weight`), линии оформлений (`text-decoration`), тень текста (`text-shadow`) и цвет текста (`color`).
```CSS
*{
   font-family: Georgia, serif; /*указываем семейство стилей и подтип*/
   font-size: 24px; /*размер можно указывать в процентах, px, em*/
   color: lightblue; /*указывать цвет можно прямым названием, шестнадцатеричным кодом, rgba и rgb*/
   font-weght:800; /*принимаются значения от 100 до 900 по сотням*/
   text-decoration: underline; /*задает подчеркивание*/
   text-shadow: 2px 3px 4px red; /*значения соответствуют - смещение по горизонатоли, по вертикали, радиус размытия и цвет тени*/
}
```
Подробнее про оформление текста можно прочитать [тут](https://html5book.ru/css3-oformlenie-teksta/).

## ОСНОВЫ CSS
Возьмем в пример самое простое стилистическое оформление и на его примере попробуем разобрать несколько различных способов применения стилей.\
\
**Первый способ** - указание для всей страницы. Делается это при помощи конструкции `*{ }` или `body{ }`. Внутрь фигурных скобок помещаем сам код.\
\
**Второй способ** - изменение стилей для определенных тегов. К примеру хотим поменять стиль у тега `<p>` - `p{ }`.
```CSS
*{
   background: lightblue;
}

body{
   background: lightblue;
}

p{
   background: lightblue;
}
```
**Третий способ** - использование классов. Для этого нам понадобится создать класс в структурном файле нашего сайта для того тега, который мы хотим видопреобразовать. Называть класс следует так, чтобы он ясно давал понять программисту для чего применяется и при этом желательно имел уникальное имя, которое не является названием тега или атрибута.\
\
**Четвертый способ** - использование идентификаторов. Делается всё аналогично классу, однако существует различие между классом и идентификатором.\
\
HTML файл:
```HTML
<div class="tekst">Текст</div>
<div id="tekst">Текст</div>
```
CSS файл:
```CSS
.tekst{ /*классы объявляются при помощи знака `.`*/
   background: blue;
}
#tekst{ /*идентификаторы объявляются при помощи знака `#`*/
   background: red;
}
```
В чем же заключается разница между идентификатором и классом?\
Идентификатор является уникальным объектом. То есть он применяется только к одному определенному блоку. Для классов такого ограничения нет.

### MARGIN И PADDING
Поговорим про главные атрибуты стилистического оформления - отступы.\
Margin и padding отвечают за внешние и внутренние отступы соответственно.\
**Внешний** - расстояния до начала блока.\
**Внутренний** - от блока до контента внутри блока.\
\
Эти два свойства могут принимать 4 значения: верхний, правый, нижний и левый.\
Если вы указываете *одно значение*, то применяться оно будет ко всем сторонам.\
Если *два значения*, то первое к верхнему и нижнему, а второе к левому и правому.\
При вводе *трех значений* - к верху, к левому и правому краю, к низу.\
Пример:
```CSS
div{
   margin: 10px 15% 1em 15%;
   padding: 15px 25px;
}
```

### DISPLAY
Данный атрибут отвечает за то, как будет отображаться ваш блок.\
Может принимать множество различных значений, про которые вы можете почитать [тут](https://developer.mozilla.org/en-US/docs/Web/CSS/display).\
Для нас же главными и самыми важными являются 4 значения: `block`, `inline-block`, `flex` и `grid`.\
`block` занимает собой всю строчку. Именно поэтому контент в нем трудно отцентровать.\
`inline-block` наоборот адаптируется под контент, который находится внутри блока, и занимает только необходимое ему пространство.

#### FLEXBOX
Если говорить простым языком, то flexbox - множество блоков внутри другого блока. Родительский блок называют контейнером. Именно для него определяют параметры отображения его дочерних элементов (`items`).

Контейнер может принимать во внимания следующие параметры: отображение (`display: flex`), направление расположений элементов внутри контейнера (`flex-direction`), перенос элементов на следующую строку (`flex-wrap`), выравнивание элементов по главной оси (`justify-content`) и по второстепенной (`align-content`).

`flex-direction` может принимать 4 значения:
- `row` - элементы располагаются слева направо (по умолчанию)
- `row-reverse` - элементы располагаются справа налево
- `column` - сверху вниз
- `column-reverse` - снизу вверх

`flex-wrap` принимает только 3 значения:
- `no-wrap` - нет переноса (по умолчанию)
- `wrap` - перенос на строку ниже
- `wrap-reverse` - перенос на строку выше

`justify-content` и `align-content` могут принимать одни и те же значения:
- `flex-start` - сдвиг элементов в начало контейнера
- `flex-end` - сдвиг элементов в конец контейнера
- `center` - отцентровка элементов
- `space-between` - равное растояние между элементами, а первый и последний находятся по краям
- `space-around` - одинаковое растоние слева и справа между всеми элементами
- `space-evently` - равное расстояние везде

Элементы же могут также изменяться по нескольким параметрам. Рассмотрим только самые важные из них: изменение порядка выстраивания элементов (`order`), увеличение элемента пропорционально другим (`flex-grow`) и выравниванивание отдельного элемента (`align-self`).

`order` по умолчанию принимает значение `1`. Чем меньше число порядка, тем ближе к началу оно становится.

`flex-grow` по умолчанию также `1`. Если же поставить значение `2`, то элемент займет в два раза больше места, чем остальные элементы.

`align-self` применяется по умолчанию значение `auto`. Однако можем поменять значения на `flex-start`, `flex-end`, `center`, `baseline` и `stretch`.

Пример синтаксиса.\
HTML файл:
```HTML
<div class="container">
	<div class="item item-1">1</div>
	<div class="item item-2">2</div>
	<div class="item item-3">3</div>
	<div class="item item-4">4</div>
	<div class="item item-5">5</div>
	<div class="item item-6">6</div>
</div>
```
CSS файл:
```CSS
.container{
	display:flex;
	flex-direction: row-reverse;
	flex-wrap: wrap;
	justify-content: center;
}
.item-2{
	order:0;
	flex-grow:2;
	align-self: center;
}
```
Для подробного изучения с визуальными примерами на Хабре есть [статья по flexbox](https://habr.com/ru/articles/467049/).

#### GRID
По сравнению с flexbox у Grid есть одно важное преимущетсво. Он может изменять размер элементов сразу по двум осям. Это и делает его незаменимым инструментом верстки сайтов.

Для начала сразу скажем, что структура CSS Grid очень схожа с flexbox. А именно имеются 2 типа блоков:
контейнер (родительский) и элементы (дочерние).

У родительского же есть обязательные свойства, без которых наши элементы не примут формат сетки:

`display: grid` - то, что нужно прописывать для контейнера в первую очередь. Работает аналогично flexbox

`grid-template-columns` и `grid-tempplate-rows` - задают количество колонок, строк, а также их размерность. Для этих атрибутов существует сокращенная запись `grid-template`. 

После прописывания этих свойств наши элементы уже примут нужный нам вид, однако их можно также продолжать кастомизировать использую следующие свойства:

`row-gap` и `column-gap` - указывает расстояние между элементами

`justify-items`, `jistify-content`, `align-content` и `align-itmes` - работают аналогично одноименным свойствам flexbox, но есть одно главное отличие. В flex-box justify отвечает за главную ось (она определяется по направлению элементов внутри контейнера), а align - за перпендикулярную. В Grid же justify всё время отвечает за ось X, а aling - за ось Y.

В свое время дочерние элементы также имеют свои свойства:\
`grid-column-start`, `grid-column-end`, `grid-row-start` и `grid-row-end` - отвечают за местоположение конкретного блока в сетке. Параметры, которые указываются внутри этих свойств должны соответствовать разделению по сетке. То есть если мы хотим вставить элемент в 3 колонку, то должны прописать что начало элемента расположено в 3 линии, а конец будет в 4.\
У этих свойств также существует сокращенная запись: `grid-row` и `grid-column`. 

Давайте теперь на примере покажем как работают и записываются все вышеперечисленные свойства.

HTML код:
```HTML
<div class="container"> <!--создание контейнера-->
	<div class="item item-1">1</div> <!--создание элементов внутри контейнера-->
	<div class="item item-2">2</div>
	<div class="item item-3">3</div>
	<div class="item item-4">4</div>
	<div class="item item-5">5</div>
	<div class="item item-6">6</div>
	<div class="item item-7">7</div>
	<div class="item item-8">8</div>
</div>
```
CSS код:
```CSS
.container{
	display: grid;
	grid-template-columns: 20% 200px 1fr; /*через пробел мы перечисляем все колонки и их размеры*/
	grid-template-rows: 100px 3fr 1fr; /*значение fr распределяется всё свободное место в равных пропорциях между друг другом*/
	row-gap: 20px; /* задаем расстояние между строками*/
	column-gap: 3%; /* задаем расстояние между колонками*/
}

.item-2{
	background: orange;
	grid-row-start: 1; /*начинаться блок будет с самого начала*/
	grid-row-end: 3; /*заканчиваться будет, где заканчивается вторая строка*/
	grid-column: 2/3; /*аналогичная сокращенная запись, чтобы вставить элемент во вторую колонку*/
}
```

Используя grid мы можем использовать и другой метод для позиционирования блоков. Данный метод называется `grid-template-areas`. Используя данную структуру мы можем прописать при помощи строчных указаний где должен стоять тот или иной элемент. 

Пример, где HTML код остается аналогичным предыдущему.
```CSS
.conteiner{
	display: grid;
	grid-template-areas: 	"a b b c"
				"d b b e"
				"f g h h";
}

	grid-area: a;
}

}

	background: yellow;
	grid-area: b;
}
.item_4{
	background: green;
	grid-area: c;
}
.item_5{
	background: green;
	grid-area: d;
}

}
```

### РАБОТА С ФОНОМ
При помощи CSS вы можете сделать фон тремя стандартными способами:
- заливка (одноцветный фон)
- градиент
- изображения

Первый способ мы уже разбирали в самом начале работы с CSS. Перейдем к изучению градиента.
#### Градиент
При задании градиента у вас есть также несколько способов это сделать:
- `linear-gradient` - стандартный линейный градиент с плавным переходом из одного цвета в другой сверху вниз
- `horizontal-gradient` - то же самое, что и линейный, но переход идет слева направо
- `radial-gradient` - круговой градиент, который распространяется по кругу (по умолчанию из центра)
- `conic-grdient` - конические градиент, аналог кругового (не затрагивали на курсе)

Сразу скажем, что второй вид бесполезный, так как его можно добиться при помощи первого с указанием угла распространения.\
Также при использовании градиентов вы можете указать столько цветовых переходжов - сколько вам требуется, не обязательно ограничиваться только двумя цветами.\
Ну и в конце концов, мы можем задать в процентах или пикселях сколько место уйдет на каждый цветовой переход.

Задание линейного градиента в CSS:
```CSS
.linear-grad{
	background: linear-gradient(135deg, red 20px, green 25%, #00F);
}
```
Таким образом первым пунктом мы указали угол распространения градиента, а последующие значения отвечают за цветовые переходы.

Теперь поработаем с радиальным градиентом. Как уже было сказано, задается он при помощи конструкции `radial-gradient`. Так как он распространяется одинаково во все стороны, то указание угла распространения тут не имеет никого смысла. Вместо этого параметра тут имеется указание точки центра градиента.

Задание кругового градиента в CSS:
```CSS
.radial-grad{
	background: radial-gradient(circle at 25% bottom, red 20px, green 25%, #00F);
}
```
Теперь первым параметорм мы указали, что распространяется он как круг с центром, находящимся снизу и 25% отступ слева.

#### Фоновое изображение
Для работы с изображениями в качестве фона существует специальная конструкция `background-image: url();`.\
Если ваше изображение имеет разрешение меньшее, чем экран браузера, то изображение будет повторяться. Данный параметр можно контролировать при помощи конструкции `background-repeat`, у которого есть следующие значения:
- `repeat` - повторение по осям x и y
- `repeat-x` - повторение только по оси x
- `repeat-y` - повторение только по оси y
- `no-repeat` - фон не дубрируется
- `round` - фон центруется, и дублируется во все стороны
- `space` - изображения повторяются с интервалом

Если вы хотите сделать фон фиксированным при скроллинге, то для этого есть свойство `background-attachment`, у которого по умолчанию стоит значение `scroll`, но можно заменить на значения `fixed` или `local`.

Позиционирование фона можно добиться при помощи `background-position`, а размер контролирует `background-size`.

Ну и напоследок рассмотрим `background-clip` - использование границ для фона. Принимает значения:
- `border-box` - фон до внешнего края блока
- `padding-box` - фон до внешнего отступа блока
- `content-box` - фон только внутри контента
- `text` - фон только для текста

Главная проблема, которая возникает при работе с изображениями в качестве фона - плохая читаемость контента на том или ином фоне. `opacity` работает не только для фона, а для всего блока, включая дочерние, поэтому просто данную проблему не решить. Для решения этой проблемы следует разобраться с `position`.`

### POSITION
`position` используется для указания расположения блока относительно общего потока других блоков. У данного свойства есть 5 значений:
- `static`
- `absolute`
- `relative`
- `fixed`
- `sticky`

`static` - значение по умолчанию. Говорит про то, что блок находится в стандартном потоке. Стоит там, где вы прописали его в HTML, а также не имеет свойств `top`, `bottom`, `left` и `right`.

`absolute` - значение, которое вытаскивает блок из стандратного потока. Остальные блоки, считают что это место пустое и пытаются его занять. Сам же блок с таким значением помещается поверх всего остального, но остается на своем месте. Позиционирование происходит относительно ближайшего позиционированного предка (по умолчанию - `body`). 

`relative` - значение, которое говорит, что для всех дочерних элементов этот блок будет являться позиционированным. Работает так же как и `static`, однако уже имеет свойства `top`, `bottom`, `left` и `right`.

`fixed` - значение, которое закрепляет блок с данным значением на экране поверх остальных и остается на том же месте при скроллинге.

`sticky` - работает примерно как `fixed`, но действует только внутри своего родительского блока. Когда родитель уходит с экрана, его покидает и блок с данным значением.

## ОСНОВЫ JAVA SCRIPT
Первый вопрос, которые возникает в начале изучения JS: "Надо ли ставить точку с запятой в конце строки?"\
Не обязательно. Обычно интерпретатор сам ставит этот символ в конец строки, но может возникать критическая ситуация, если следующая строка начинается с цифры или со специального знака. На нашем курсе мы не будем начинать строки с таких знаков, так что и точку с запятой в коцне строки использовать не будем.

В JS есть несколько способов инициализации переменной. На самом старте данного языка была только конструкция `var`. Однако в 6 версии языка появились еще две дополнительные - `let` и `const`. Именно две последние и советуют использовать новые спецификации, так как `var` не самая надежная штука из-за ее области видимости и возможности переопределения. Различие между `let` и `const` одно - можно или нельзя изменять переменную.
